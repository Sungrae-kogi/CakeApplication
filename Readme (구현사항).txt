[구현내용]

1. 로그인 화면
- RelativeLayout 을 사용하여 위젯(EditText, Button)을 수직하게 배치
- Activity 를 사용하여 구현함
- 첫 실행 시 RelativeLayout 에 invisible 로 배치된 접속 페이지의 상태를 visible 로 바꿔서 화면 전체에 보여줌
- 아이디, 비밀번호를 입력하기 위한 EditText 2개를 배치하고 빈 입력인지, 아이디와 비밀번호가 존재하는지 확인함
  이상이 있을 시 Toast 로 유저에게 알리고, 이상이 없으면 로그인 처리하고 메인 화면으로 이동함

2. 회원가입 화면
- LinearLayout 을 사용하여 위젯(EditText, Button)을 수직하게 배치
- 다음 경우에 회원가입이 실패함
  a. 빈 입력이 있음
  b. 비밀번호와 비밀번호 재입력 값이 불일치
  c. 아이디가 3글자 미만
  d. 비밀번호가 4글자 미만
- 회원가입 버튼 클릭 시 대화상자로 약관을 보여줌. Decline 을 선택하면 회원가입 거부
- 약관을 동의하면 회원가입 처리되고 로그인 화면으로 이동함

3. 메인 화면
- ConstraintLayout 을 이용하여 리사이클러 뷰와 버튼을 화면에 배치
- 케이크의 사진과 이름을 포함하는 아이템을 리사이클러 뷰에 리스트로 보여줌
- 추가 버튼을 누르면 케이크가 추가되고, 삭제 버튼을 누르면 케이크가 삭제됨
- 회원정보를 누르면 대화상자 형식으로 회원정보를 보여주거나 회원가입을 할 지 물어봄

4. 자료형
- 케이크를 표현하기 위해 Product 클래스를 구현함. 문자열 title (이름), 정수형 resource (Drawable 리소스 아이디)를 가짐
- 회원정보를 표현하기 위해 User 클래스를 구현함. 아이디, 비밀번호, 이름, 전화번호, 주소 필드를 가짐

5. 회원가입/로그인 처리
- Application 클래스를 상속한 App 클래스를 정의하여 앱 전역에서 접근 가능하도록 함
- 회원가입 시 회원 정보를 User 객체로 생성해 App.register() 를 호출
- 현재 회원 정보 확인 시 App.getCurrentUser() 를 호출. 로그인 되어 있지 않으면 NULL 이 리턴됨
- 로그인 시 App.login() 을 호출함
